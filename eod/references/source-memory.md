# Memory Source

Use this module for prior decisions, recurring project context, and narrative enrichment.

Memory is not authority. Treat it as a way to find likely repos, branches, PRs, worktrees, blockers, and preferences, then verify against current local files or GitHub.

## Use When

Use memory by default for EOD generation when available, especially for:

- configured repo and worktree routing
- recurring project context
- prior unresolved blockers
- preferred verification commands
- recently discussed PRs or issue batches

## Extract

Capture:

- likely repo or worktree paths
- issue or PR references to verify
- previous decisions that explain today's work
- user preferences that affect wording or next steps

## Verification Notes

Every PR or issue found in memory must be added to the verification queue. Verify with GitHub before inclusion.

If memory is stale or conflicts with local git, current conversation, or GitHub, trust the current source.
